# 전략 배치 생성 (유튜브 스크립트 → 백본 전략 다량 생성) 스펙

작성일: 2026-07-17. 사용자 요청: 유튜브 스크립트 여러 개를 한 번에 넣으면, 각각을 분석해서
백본 전략(파라미터 미세튜닝의 대상이 될 원본)을 다량으로 만들어주는 모듈. 목적은 두 가지 —
① 튜닝 대상이 될 강력한 백본 자체를 늘리는 것, ② 엔진이 RSI/거래량/볼린저 등 지표를 실제로
"이해"하고 있음을 보여줘 사용자가 전략을 직접 수정할 때 참고가 되게 하는 것(교육적 가치).

## 1. 기존 자산 재사용 (새로 만들 필요 없는 것)

- `core/nl_strategy.py::interpret_strategy_text(raw_text)` — 스크립트 1개를 해석해 flat/staged
  스키마 중 알맞은 쪽으로 자동 판별(`_looks_like_staged_strategy`)해 반환. RSI/볼린저/MACD/
  일목균형표/캔들패턴(마루보즈·핀바·도지·인사이드바·쌍바닥/쌍봉·다이버전스·적삼병 등)까지 이미
  지원 — 지표 커버리지는 매우 넓다. `description` 필드에 "왜 이렇게 해석했는지" 한국어 설명이
  이미 포함됨(교육적 목표 ②는 기존 함수가 이미 충족).
- `app/pages/1_백테스팅.py`의 "🤖 자연어 전략 등록" 탭 — 단건 처리 UI 참고용(job_manager 패턴).
- `core.backtest_engine.compare_with_benchmarks` — 생성된 config 하나를 특정 종목/기간에 즉시
  백테스트해 성과 지표(샤프/거래횟수 등)를 얻는 기존 함수. sanity 필터에 그대로 재사용.
- `core.strategy_tuning.sample_universe` — 섹터 균등 표본 추출(sanity 백테스트용 종목 몇 개 뽑기).

## 2. 실제로 비어있던 갭 (이번에 메울 것)

`core/nl_strategy.py`의 `STAGE_CONDITION_PROPERTIES` enum + `STAGED_SYSTEM_PROMPT`에
`volume_spike`/`volume_dryup`가 빠져 있음 (엔진 `core/strategy_engine.py`에는 이미 구현돼 있고
`core/indicators.py::compute_volume_ratio`/`compute_volume_dryup_ratio`도 존재) — "거래량 터지면
진입", "거래량 마르면 청산" 같은 설명이 나와도 지금은 인식할 방법이 없었다. 두 지표를 enum과
프롬프트 설명에 추가한다 (period/lookback 파라미터는 각각 volume_spike=period(기본20),
volume_dryup=lookback(기본10) — 기존 STAGE_CONDITION_PROPERTIES의 같은 이름 필드를 공유해서 씀).

## 3. 신규: 배치 생성 페이지

`app/pages/15_전략_배치_생성.py` (신규).

- **입력**: 텍스트 영역 1개에 여러 스크립트를 구분자(`\n---\n`)로 이어붙여 붙여넣기. 스크립트
  개수는 설계상 5~15개를 기준으로 하되 상한을 강제하진 않음(사용자가 Gemini 무료 티어 한도를
  스스로 감안).
- **처리**: 각 스크립트를 `interpret_strategy_text()`로 순차 해석(job_manager로 백그라운드 실행 +
  진행률 표시 — Gemini 호출 특성상 스크립트당 수 초 소요). 실패(파싱 오류/빈 응답)한 스크립트는
  결과 테이블에 "해석 실패"로 남기고 건너뜀.
- **자동 sanity 백테스트**: 해석에 성공한 config마다, 섹터 균등 표본 5종목 × 최근 3년으로
  `compare_with_benchmarks` 실행. PASS 기준(둘 다 만족해야 함):
  - 5종목 합산 거래횟수 ≥ 3 (전혀 진입을 못 하는 죽은 조건 제거)
  - 5종목 평균 샤프가 NaN이 아님(계산 자체가 성립)
  기준을 통과 못 하면 "sanity 실패"로 표시하되 결과에서 숨기지 않음(사용자가 원하면 그래도 저장
  가능하게 — 자동 필터링은 참고용 신호이지 강제 차단이 아님).
- **결과 테이블**: 스크립트 순번, 생성된 전략명, PASS/FAIL, 5종목 평균 초과수익, 합산 거래횟수,
  해석 설명(expander로 원문 대비 해석 근거).
- **저장**: 체크박스로 원하는 결과만 선택 → "선택 저장" 버튼으로 `Strategy` 테이블에 등록
  (source="배치생성", description에 sanity 지표 요약을 덧붙임). **자동으로 튜닝 큐에 편입하지
  않음** — 사용자가 "전략 관리"에서 확인 후 직접 야간 미세튜닝 대상으로 고르는 기존 흐름을 그대로
  따른다(사용자 확정: 자동 편입 아니라 라이브러리 저장 후 수동 선택).
- **지표 해설 섹션**: 이번 배치에서 실제로 쓰인 지표 종류를 모아, 지표별 한 줄 설명(엔진 프롬프트에
  이미 있는 설명 문구 재사용)을 보여주는 expander 하나 추가 — 사용자가 왜 이 조건이 선택됐는지
  RSI/거래량/볼린저 등을 반복 노출로 익히게 하는 보조 장치(사용자가 밝힌 학습 목적에 직접 대응).

## 4. 범위 밖 (이번에 안 함)

- 여러 스크립트를 하나로 합성/재조합(예: 스크립트 A의 진입 + 스크립트 B의 청산)하는 기능 — 이미
  1_백테스팅.py "🧩 전략 합성" 탭이 존재하므로 중복 방지, 필요해지면 그쪽을 확장.
- 생성 전략의 야간 튜닝 큐 자동 편입 — 향후 요청 시 별도로 논의.
- 대량(수십~100개) 배치의 job_manager 큐잉/재시도 UI — 이번 스펙은 5~15개 기준(사용자 확정).
