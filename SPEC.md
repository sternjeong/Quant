# 개인용 주식 도우미 사이트 — 프로젝트 스펙 (v1.0)

> 대화를 통해 확정된 스펙입니다. 구현 중 세부사항이 바뀌면 이 문서도 함께 갱신합니다.

## 0. 개요

- **사용자**: 나 혼자 (로그인/보안 불필요)
- **실행 환경**: 내 컴퓨터에서 로컬 실행 (localhost)
- **대상 시장**: 미국 주식만 (S&P500, 개별 종목)
- **핵심 목적**: 유튜브 등에서 본 매매 전략을 과거 데이터로 검증하고, 검증된 전략을 관심 종목 50개에 매일 적용해 타점 발생 시 알림을 받는 것

## 1. 기술 스택 (제안)

| 영역 | 선택 |

| 언어 | Python |

## 2. 모듈 구성

### 모듈 A. 백테스팅 엔진

- **비교 대상**: 특정 전략 적용 vs S&P500 매수 후 보유 vs 개별 종목 그대로 홀딩
- **전략 템플릿** (전부 포함, TradingView 스타일 UI로 지표를 버튼 클릭으로 on/off):
  - 이동평균 교차 (골든/데드크로스, 단기·장기 기간 조절 가능)
  - RSI 과매수/과매도
  - 볼린저밴드 (상단/중심선/하단 이탈)
  - → 추후 지표 추가 및 여러 지표 AND/OR 조합 가능하도록 엔진 설계
- **볼린저 밴드 응용 전략 4종 + 진입가 기준 손절 (신규, 2026-07-15)**: 스퀴즈(밴드폭 축소 후 확대+
  밴드 돌파)/추세추종(%B+MFI)/추세반전(쌍바닥·쌍봉)/다이버전스(가격-RSI) — 전부 1:2:6 staged
  스키마(`entry_stages`/`exit_stages`)를 1단계짜리로 활용해 표현(진입≠청산 조건이라 레짐 스키마로는
  불가능). staged 스키마에 `stop_loss`(진입 시점 가격 레벨을 스냅샷해 고정하는 손절, source=
  중심선/최근 저점/최근 고점) 신규 지원 — 4개 전략 모두 "진입 시점 기준" 손절을 요구해 일반 기능으로
  구축. 엔진은 롱온리라 매도(숏) 방향은 미구현. 설계 배경은 `BOLLINGER_STRATEGIES_SPEC.md` 참고.
- **직접 수식 입력 슬롯 (신규)**: 지표 토글로 표현하기 어려운 조건은 사용자가 파이썬과 비슷한
  문법의 불리언 수식을 직접 입력해 전략을 만들 수 있음 (예: `close > sma(close, 20) and
  rsi(close, 14) < 30`). eval() 대신 ast 화이트리스트 인터프리터로 안전하게 평가
  (core/expression_engine.py). 저장/불러오기/전략 관리 화면에서 지표 토글·1:2:6 단계별 전략과
  동일하게 취급됨
- **결과 지표**: 아래 전부 계산해두고, 사용자가 화면에 표시할 지표를 선택하는 방식
  - 누적수익률 / CAGR
  - MDD (최대낙폭)
  - 샤프지수
  - 승률 / 매매 횟수
- **UI 방향**: TradingView 차트처럼 캔들 위에 지표를 오버레이하고, 지표별 토글 버튼 제공
- **백테스팅 기간**: 시작일/종료일을 직접 지정 가능
- **자연어 전략 등록 (신규)**:
  - 유튜버 등의 전략 설명 스크립트(텍스트)를 붙여넣으면, AI가 먼저 "이 전략은 이런 조건입니다"라고 해석해서 보여줌
  - 해석된 조건을 기존 지표 엔진(이동평균/RSI/볼린저밴드 및 향후 추가 지표)의 조합으로 매핑
  - 전략명은 유튜버 설명 기반으로 자동 생성(예: "골든크로스+RSI 눌림목") 또는 "후보1"처럼 임시명 부여 후 저장
  - 저장된 전략은 전략 라이브러리에 누적되어 언제든 다른 기간/종목으로 재실행 가능
- **다종목 미세튜닝 (신규, 2026-07-14 추가)**:
  - 백본 전략(유튜버 해석 결과 또는 라이브러리 저장 전략)을 S&P500 섹터 균등 표본(기본 100종목)에
    한 번에 적용해, 종목이 6개 스타일(주도주/성장주/가치주/경기민감주/경기방어주/퀄리티 컴파운더)
    중 어디에 속하는지 판별하고 그 방향에 맞게 파라미터를 자동 탐색(grid/random search)한다
  - 전략의 지표 구성/조건 로직(백본)은 그대로 유지하고 수치 파라미터만 탐색(강화학습 미사용)
  - train(과거)/test(최근, out-of-sample) 기간 분리로 과최적화 방지, 종목별로 전략 적용 vs 종목
    매수보유 vs S&P500 매수보유 3-way 비교 결과를 제공(기존 3-way 비교 로직 재사용)
  - 사용자가 반기(6개월) 주기로 재실행할 것을 전제로, 매 실행을 새 배치 레코드로 영구 저장(장기
    이력 누적). 상세 설계는 `STRATEGY_TUNING_ENGINE_SPEC.md` 참고, 구현은 `core/strategy_tuning.py`
    + `app/pages/1_백테스팅.py`의 "🧬 다종목 미세튜닝" 탭
- **전략 합성(신규, 2026-07-15 추가)**:
  - 전략 라이브러리에 저장된 서로 다른 두 전략을 골라 신호를 AND(둘 다 보유 신호일 때만 진입)/
    OR(하나라도 보유 신호면 진입)로 합쳐 새 전략을 만든다: `{"combine": "AND"|"OR", "strategies":
    [<하위 전략 A>, <하위 전략 B>]}` — 네 번째 indicator_config 스키마(레짐/직접 수식/1:2:6
    단계별에 이어)로, `core/strategy_engine.py::evaluate_boolean_signal()`이 4종 전부를 판별해
    "포지션 보유 중" 불리언 시그널로 통일한다
  - 하위 전략은 레짐/직접 수식/1:2:6 단계별/전략 합성 어떤 스키마든 재귀적으로 담을 수 있음(전략
    합성끼리도 다시 합성 가능). 1:2:6 단계별 하위 전략은 비중(weight)>0인 구간을 보유로 간주해
    불리언으로 단순화
  - 합성 결과는 일반 전략과 동일하게 전략 라이브러리에 저장되어 백테스트 재실행/관심종목 연결이
    가능하다(단, 다종목 미세튜닝 대상으로는 아직 지원하지 않음 — 파라미터 튜닝은 레짐/직접 수식/
    1:2:6 단계별 백본에만 적용됨)
  - 구현은 `app/pages/1_백테스팅.py`의 "🧩 전략 합성" 탭 (기존에 미구현 상태로 방치되어 있던
    "🔬 알고리즘 자동 생성" 빈 탭 자리를 재활용함)
- **캔들스틱 패턴 지표 9종 + 캔들 패턴 매매 전략 9종 (신규, 2026-07-15)**: 마루보즈/핀바/도지(4종)/
  인사이드바(+돌파 이벤트)/관통형(피어싱·흑운형)/모닝스타·이브닝스타/적삼병·흑삼병/삼법형을
  `core/indicators.py`에 지표 함수로 구현(수학적 정의는 `CANDLESTICK_PATTERNS_STRATEGY_SPEC.md` 참고).
  영상 원문에 손절/익절까지 명시된 5개(강세 핀바 반전/상승 장악형 돌파/상승 관통형(볼린저 밴드)/
  모닝스타 반전/상승 삼법형(20 EMA))는 그대로, 정성적 설명만 있던 4개(마루보즈 돌파/잠자리형 도지
  반전/인사이드바 돌파/적삼병 상승 지속)는 웹 검색으로 표준 진입·손절·익절 관례를 보충해 총 9개를
  전략 라이브러리에 `source="youtube_script"`로 직접 등록. staged 스키마에 손절 대비 배수 익절
  (`take_profit`, stop_loss와 함께 정의)과 저항/지지선 돌파(`level_break`)·단일 이동평균선 터치
  (`ma_touch`) 신규 지원. 롱온리 제약상 각 패턴의 숏(bearish) 워크스루는 전략 미등록(지표 함수 자체는
  양방향 구현). 상승 삼법형은 실제 시장에서 극히 드문 고전 패턴(Bulkowski 실증 연구: 캔들 470만 개 중
  102개만 발견)이라 단일 종목 짧은 기간 백테스트에서 매매 0건이 나오는 것이 정상 동작임을 확인.

### 모듈 B. Threads 글 → 티커별 요약

- Threads 크롤링이 막히면 → 텍스트를 수동으로 복사해서 붙여넣는 입력창 제공
- AI가 본문을 분석해 관련 티커를 **자동 인식**
- 자동 인식이 틀렸을 경우 **내가 직접 티커를 수정/재태깅**할 수 있는 기능 포함
- 티커별로 요약 히스토리 누적 저장 (같은 티커에 대한 여러 글을 시간순으로 모아보기)

### 모듈 C. 관심 티커 리스트 (최대 50개) + 매일 타점 모니터링

- 백테스팅에서 검증된 전략을 종목별로 저장
- 스케줄러가 매일 미국 장마감 후 50개 종목을 스캔 → 저장된 전략 조건 충족 시:
  - 웹 대시보드에 타점 표시
  - 데스크톱 알림 발송

### 모듈 D. 거장 포트폴리오 추종 (신규 — 영상 참고 후 추가 제안)

- SEC EDGAR 13F 공시(무료, 분기 지연 공시)를 파싱해 유명 펀드매니저의 보유 종목 추적
- 여러 거장을 관심 목록에 등록하면, 그들이 **공통으로 매수한 종목** 교집합 표시
- **기본 추적 거장 리스트 (확정)**:
  - 워런 버핏 (버크셔 해서웨이)
  - 마이클 버리 (사이언 애셋 매니지먼트)
  - 빌 애크먼 (퍼싱스퀘어)
  - 캐시 우드 (ARK — 13F 외 일별 보유내역도 공개되어 더 정밀 추적 가능)
  - 스탠리 드러켄밀러 (듀케인 패밀리오피스)
  - 데이비드 테퍼 (아팔루사)
  - 세스 클라만 (바우포스트)
  - (추후 관심 있는 다른 펀드매니저 자유롭게 추가 가능)
- **ETF/액티브펀드 구성종목 조회 (신규)**:
  - 미국 ETF: 운용사(iShares, SPDR 등)가 공개하는 holdings CSV를 자동 연동
  - 국내(한국) ETF/펀드: 무료 API가 마땅치 않아 CSV/엑셀 파일을 직접 업로드하면 파싱해서 보여주는 방식으로 지원

### 모듈 E. 퀀트 스크리너 (신규)

- PER, PBR, 시가총액, 섹터, 기술적 지표 등 조건으로 미국 주식 전체를 필터링
- 스크리닝 결과를 관심 티커 리스트(모듈 C)에 바로 추가 가능

### 모듈 F. 밸류에이션 도구 (신규)

- PER/PBR 히스토리 밴드 차트
- 동종업계 피어 비교 테이블 (멀티플 비교)
- 여러 밸류에이션 방법론(DCF, DDM, PER/PBR 상대가치, EV/EBITDA, PEG, Graham Number 등)을 최대한 구현
- 특정 방법론으로 좁히지 않고, **한 화면에서 여러 기법을 동시에 비교하며 직접 판단**할 수 있는 UI(탭/카드 형태)에 집중

### 모듈 G. 매크로 대시보드 (신규)

- FRED API 기반 경제지표 캘린더 및 히스토리 (금리, GDP, 물가, 실업률 등)
- 경기 사이클 국면 판단 + 섹터 로테이션(국면별 아웃퍼폼 섹터) 시각화

### 모듈 H. 포트폴리오 관리 (포함 확정)

- 실제 보유 종목/수량/매입가를 입력해서 손익, 리스크(변동성, 상관관계, 섹터 집중도) 분석
- AI가 포트폴리오에 대한 코멘트 생성

### 모듈 I. 차트 조회 (신규 — 2026-07-12 추가)

- 백테스팅/전략과 무관하게, 티커만 입력하면 바로 캔들차트를 볼 수 있는 조회 전용 화면
- 봉 주기: yfinance가 지원하는 범위 전부(1분~90분 분봉/시간봉, 일봉, 5일봉, 주봉, 월봉, 분기봉)
  - 분봉/시간봉은 yfinance 자체 제약으로 조회 가능한 과거 기간이 짧으므로(예: 1분봉 최근 6일,
    5~30분/90분봉 최근 60일, 60분봉 최근 730일), 사용자가 더 오래된 시작일을 고르면 자동으로
    당겨서 보정하고 화면에 안내
- 캔들 위에 이동평균/볼린저밴드/일목균형표 구름대를 오버레이하고, RSI/MACD는 하단 별도 패널로
  토글 on/off (모듈 A 백테스팅 엔진과 동일한 지표 계산 로직 재사용)
- 거래량 바 차트 항상 표시
